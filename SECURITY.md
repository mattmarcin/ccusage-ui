# Security Policy

## Supported Versions

This project is pre-1.0. Security fixes are handled on the default branch until a formal release policy exists.

## Reporting a Vulnerability

Please report security issues privately by opening a GitHub security advisory on the repository, or by contacting the maintainer directly if advisories are unavailable.

Do not include private `ccusage` JSON output, local usage logs, API keys, or other credentials in public issues.
